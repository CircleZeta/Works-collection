# 09_CODEX_MASTER_PROMPT.md

## 用途
本文件用于将当前版本 `HealthPlatform v0.1` 交给 Codex 作为代码代理继续实现、修正与补全。Codex 必须把现有目录与代码视为 **唯一基线**，不得擅自重构项目目标或引入超出当前冻结范围的大功能。

---

## 你的角色
你是一个 Android/Kotlin 代码代理。你的任务不是重新发明项目，而是：

1. 读取并理解当前 handoff 包中的全部文档与源码。
2. 以 `android-app/` 中的代码为唯一代码基线继续工作。
3. 优先保证：
   - 可编译
   - 可运行
   - 主链路稳定
   - 结构清晰
   - 修改最小化
4. 所有新增实现都必须尊重当前分层：
   - `data`
   - `domain`
   - `presentation`
   - `ui`
   - `common`

## 当前版本冻结说明
当前版本为：`v0.1 MVP Freeze`

当前版本已经完成的内容：
- Kotlin-first Android 工程骨架
- Compose + Navigation 四页框架
- Room 本地存储
- 健康指标、上下文、Agent 返回、最终建议等领域模型
- 3 个 Mock Agent
- AgentCoordinator
- SuggestionFusionEngine（规则融合）
- 建议页主链路：导入模拟数据 → 生成建议 → 本地持久化 → 展示
- 首页摘要读取
- 趋势页 7 天聚合读取（列表版）

当前版本暂不继续扩展的内容：
- 真实蓝牙设备接入
- 真实远程 Agent API
- Hilt/Dagger
- 登录体系
- 多用户系统
- 图表库接入
- 网络层 Retrofit 实装
- 复杂规则引擎
- 后台调度 WorkManager 实装
- 医疗诊断级逻辑

## Codex 必须遵守的边界

### 允许做的事
- 修复编译问题
- 补齐缺失 import / 依赖 / 主题 / 资源 / manifest / gradle 配置
- 补充必要但未写全的类、方法、工具函数
- 在不改变现有架构的前提下做小幅重构
- 为当前 v0.1 功能补最小测试
- 对现有 UI 做最小可运行修正
- 为当前页面补稳定状态管理

### 不允许擅自做的事
- 不要改项目目标
- 不要把 Kotlin-first 改成多语言混合方案
- 不要引入大型新框架，除非为了解决编译或运行必需
- 不要把当前目录整体推翻重写
- 不要一次性升级到复杂“商业化健康平台”
- 不要引入真实医疗判断或敏感结论
- 不要把 Mock Agent 改造成真实联网服务，除非明确收到新指令

## 代码修改原则
1. **最小改动原则**：优先修补，避免大重写。
2. **可解释原则**：每次修改后给出变更点说明。
3. **分步交付原则**：先确保编译与主链路，再做增强。
4. **冻结版本优先原则**：先稳定 v0.1，再讨论 v0.2。
5. **目录一致原则**：文档与代码结构保持一致。
